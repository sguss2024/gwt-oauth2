This library allows your app to easily interact with an [OAuth 2.0 provider](http://wiki.oauth.net/w/page/25236487/OAuth-2). It allows your users to grant access to your app to view and modify private data on their behalf using the OAuth 2.0 protocol.

The **[sample application](http://gwt-oauth2.googlecode.com/svn/trunk/samples/multi/demo/Sample.html)** ([see the code](https://code.google.com/p/gwt-oauth2/source/browse/trunk/samples/multi/com/google/api/gwt/oauth2/samples/multi/client/OAuth2SampleEntryPoint.java)) demonstrates getting an access token with:

  * [Google](http://code.google.com/apis/accounts/docs/OAuth2.html)
  * [Instagram](http://instagram.com/developer/auth/)
  * [Foursquare](https://developer.foursquare.com/docs/oauth.html)
  * [Facebook](http://developers.facebook.com/docs/authentication/)
  * [Dailymotion](http://www.dailymotion.com/doc/api/authentication.html)
  * [Windows Live](http://msdn.microsoft.com/en-us/library/hh243647.aspx)

...though this library works with any OAuth 2.0 provider that supports the JavaScript flow.

The [Getting Started](https://code.google.com/p/gwt-oauth2/wiki/GettingStarted) guide describes using OAuth 2.0 with Google as the provider, though, again, the concepts should be the same for all OAuth 2.0 providers.

You can also read the library's [JavaDoc](http://gwt-oauth2.googlecode.com/svn/trunk/javadoc/latest/index.html).
